## D1 — The system under evaluation
The system whose expected behavior Δ measures is the **deployed model** (population-level
causal expectation over previous sessions of the same model). Sessions are trajectories
*of* the system, not systems themselves.

## D2 — Observable stream and expectation (O_D)
- Event = token. ω = surprisal (−top1 logprob). Entropy and top-1 gap are **not kernel
  observables**; they appear only as mandatory comparison baselines under D6.
- ω̂ = strictly causal per-position-bucket mean over previous sessions of the same model;
  buckets [0,8,16,32,64,128,256,512,1024,2048,4096]; warm-up min_sessions = 5.
- Reproducible synthetic C3 record: **PASS, r_Δω = −0.09194754 (≈ −0.092)**, generated by
  `examples/make_synthetic.py structural` with seed 7, 120 sessions,
  `prama-protokol==0.1.0`, CFG(tau_memory=64, g_smooth=16), K=256, min_sessions=5.
  C3 will be re-run and committed on the real stream before unblinding.

## D3 — Kernel configuration and bin scale (the AS-1 gap, now with data)
AS-1 C5 fixes kernel parameters across domains *in bins* without defining bin scale.
This study declares: **bin = token**.

**Empirical finding (synthetic pipeline validation, 2026-07, pre-outcome):** the
validated grid configuration (tau_memory = 336, g_smooth = 24) is **mechanically
inapplicable** at K = 256 token windows — the accumulator's memory exceeds the
evaluation trajectory, so Ξ never reaches regime (latent AUC collapses to 0.22).
A tau × g_smooth sweep shows a **robust plateau at tau ∈ [16, 64]** (AUC 0.90–0.98
for every g_smooth ∈ {8, 16, 24}), collapsing for tau ≥ 128. The choice is a regime,
not a knife-edge — which is what distinguishes a scale declaration from retro-fitting.

**Declared configuration:** tau_memory = 64, g_smooth = 16 (on the plateau; all other
parameters at validated values: lambda_eq 1.0, lambda_recovery 0.005, lambda_min 0.1,
theta_scale 2.0, kappa 0.05). The validated 336/24 configuration will be run alongside
on the real stream as declared sensitivity, its mechanical failure documented.

**Flagged for AS-1 v1.1:** proposed "bin-scale declaration" clause — kernel parameters
are fixed in bins *given a declared bin scale satisfying tau_memory ≪ trajectory
length*; deviations from validated values must sit on a demonstrated robustness plateau
and be pre-registered.

## D4 — Evaluation window and inclusion
- Latent occupancy computed on the first K = 256 tokens of each session (valid rows only).
- Sessions with fewer than 32 valid tokens in the window are excluded (count reported).

## D6 — Analysis, baselines, and success criteria
- **Single preregistered primary:** `latent_occupancy`, with no weighted or learned
  composite. `delta`, `xi`, and `neg_M` are separate exploratory channels.
- Primary metric: AUROC of latent-collapse occupancy for Y_o. Threshold is calibrated
  on train at target FPR 0.10 and applied unchanged to test.
- Exactly four mandatory baselines on the same causal K window:
  1. `mean_surprisal`, arithmetic mean of −top1_logprob;
  2. `mean_entropy`, arithmetic mean of per-token normalized Shannon entropy computed
     from the returned top-logprob candidates after renormalizing their probabilities
     (natural-log entropy divided by ln m, where m≤5 is the number of valid candidates);
  3. `negative_mean_top1_gap`, the arithmetic mean top-1 minus top-2 logprob gap,
     multiplied by −1 so that greater means worse for every score;
  4. `markovian_surprisal_tau64`, defined by I(0)=0 and
     I(t)=a·I(t−1)+(1−a)·ω(t), a=exp(−1/64), for t≥1.
  The fourth baseline uses the kernel's same memory scale, isolating the contrast between
  a Markovian EMA of raw activity and the Protokol's non-Markovian structural machinery.
  No additional baseline enters the confirmatory verdict.
- Null: label-permutation (1000 shuffles), threshold p < 0.01.
- Split: deterministic stratification with seed 1337 and test fraction 0.25, requiring
  both classes in train and test.
- **Power gate before score inspection:** test must contain at least 15 positive examples.
  If it does not, the run is `underpowered`, not a null. Without inspecting any score,
  collect fresh confirmatory sessions in fixed blocks of 100 under the unchanged protocol,
  recompute the deterministic split, and repeat until the gate passes or total collected
  N reaches 1000. Failure to reach 15 test positives at N=1000 terminates as
  `underpowered`, still not as an honest null.
- **Success (confirmatory):** C3 passes; latent AUROC exceeds the permutation null; and
  latent occupancy exceeds every baseline both by test AUROC and by test TPR at the
  train-calibrated target FPR 0.10.
- **Honest null:** signal present but not exceeding baselines → published as a negative
  result (ER-style).
- **Interface failure:** any failure in the frozen analysis interface before a
  confirmatory verdict is emitted, including D7 collection identity failure, preflight
  outcome/power failure, scoring/validation/input-wiring failure, or C3 failure on the
  real stream → the diagnosis is O_D, the study is invalid, a corrected interface opens
  a NEW study citing this one. The kernel is not touched.
- **Authoritative analysis artifact:** `scripts/analyze_ep1.py` alone emits the D6 gates
  and terminal verdict. Outputs from exploratory scripts cannot establish the claim.
- **Terminal verdict:** positive / honest null / interface failure. `underpowered` is a
  pre-verdict stopping state governed by the fixed extension rule above.
